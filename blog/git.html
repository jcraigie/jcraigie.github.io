Version Control

1/3/2016

Ever do something and immediately wish for a do over? Was it in a video game? No? Then you were probably out of luck. With Version Control via Git and Github you can bring the save point into a small part of your life; along with easy collaboration, organization and backup. With Version Control done properly progressive saves of the code you are working on are created by the user, allowing one to roll back to a known good save when that great idea turns bad..

Additionally with the branching architecture of Git you have a safe environment to work on your code while not easily endangering your original code. Once your new code is ready to be merged back into the master code that too is easily done with just a few command line entries.

Through regular commits(saves) pushed to Github(saved online) your code, and those save points, are not only saved for later use but also available anywhere you have internet access and are also immediately available to any one you wish to share your code with. Conversely through Github you would have easy access to revisions made by anyone you are collaborating with no matter where you, or they, are.
